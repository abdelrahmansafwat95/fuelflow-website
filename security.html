For procurement & IT

Trust, by design.

FuelFlow handles money, identity, and fuel. Here's the architecture we built to keep all three safe — written so your security review actually finishes.

Authentication Phone OTP

SMS OTP for every login — 6 digits, bcrypt-hashed in storage, 5-minute TTL. Brute-force lockout kicks in after 5 wrong attempts within 15 minutes. No passwords means no password reuse.

Sessions JWT + rotation

Short-lived JWT access tokens (15 min), long-lived refresh tokens with server-side rotation. Compromised refresh tokens are revoked on next use thanks to reuse detection. Logout is server-authoritative.

Data protection TLS 1.2+ · AES-256

TLS 1.2+ enforced on every external endpoint, with HSTS preload. PII at rest is encrypted with AES-256-GCM. Passwords use bcrypt (cost 10). Database backups are encrypted with separate keys.

Transaction security 6-layer fraud

Every authorization request hits all six rules in parallel:

  • Velocity — too many fills in too little time per vehicle or driver
  • Geo-jump — physically impossible distance between consecutive stations
  • Duplicate — same pump + same vehicle within a short window
  • Excessive litres — amount above the vehicle's tank capacity
  • Hourly cap — usage outside expected operating hours
  • New vehicle — extra scrutiny on the first transactions for a newly added vehicle

Budget integrity Atomic locks

Budget reservations use Redis distributed locks to prevent double-spend across concurrent authorizations. The actual deduction is committed inside an atomic Postgres transaction. No race conditions, ever.

Audit trail Immutable

Every entity change — budget, vehicle, driver, station, role, price — writes an audit log entry with actor, timestamp, and full before/after. Audit logs are append-only and retained for compliance review.

Access control RBAC

Database-driven, per-role, per-feature permissions. Configurable by super-admin only. Read-only roles are first-class. Two-person rule available on sensitive operations like bulk budget changes.

Egypt compliance CBE-aligned

Architecture aligned with Central Bank of Egypt fintech guidelines. Data residency in-country. Tax-compliant invoice generation. Audit logs retained per local requirements.

Responsible disclosure security@
